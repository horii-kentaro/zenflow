C-3. ユーザーフローのタイムライン

典型的な1日の利用フロー

初回起動
オンボーディング（3スライド）
アプリの主要機能を紹介するスライド → 「始める」タップ → SecureStore に完了記録 → ダッシュボードへ
朝（起動時）
ダッシュボード確認
「おはようございます」時間帯挨拶 → 今日のメッセージ表示 → クイックアクション3ボタン（未完了状態）→ タブに赤ドットバッジ表示
朝（1分）
気分記録
5段階スコア選択 → コンテキストタグ（仕事/健康/睡眠等）選択 → 保存 →
ダッシュボードのクイックアクションにチェックマーク → タブの赤ドット消滅
朝（3〜5分）
セルフケアルーティン
ルーティン種類選択（呼吸法/ストレッチ/マインドフルネス/ボディスキャン/おまかせ）→ タイマー開始 → ステップ進行 → 完了記録
Pro: AIパーソナライズドルーティン推薦カード表示
夜（3〜5分）
AIジャーナリング
おすすめトピックから選択 or 自由入力 → Claude API による対話 → AI が共感・質問・アドバイス → ジャーナル保存
週末
振り返り（Pro限定）
AI週間レポート: 平均気分スコア・記録日数・セルフケア回数 → AIインサイト3枚（感情パターン/ストレス要因/推奨アクション）
月間ムードマップ: カレンダー上にカラードットで気分推移を可視化

C-4. データモデル（Prisma スキーマ）

prisma/schema.prisma から抽出した主要モデル

// prisma/schema.prisma — 主要モデル model User { id String @id @default(cuid()) name String? email String @unique hashedPassword String emailVerified DateTime? notificationPrefs Json? // {paymentNotify: bool, weeklyReport: bool} subscription Subscription? moodEntries MoodEntry[] selfcareRecords SelfcareRecord[] journals Journal[] } model Subscription { id String @id @default(cuid()) plan String // "free" | "premium" stripeCustomerId String? stripeSubId String? status String // "active" | "canceled" | ... userId String @unique } model MoodEntry { id String @id @default(cuid()) score Int // 1〜5 note String? context String? // "work" | "relationship" | "health" | ... date DateTime @db.Date userId String } model SelfcareRecord { id String @id @default(cuid()) type String // "breathing" | "stretch" | "mindfulness" | "bodyscan" duration Int // 秒数 date DateTime @db.Date userId String } model Journal { id String @id @default(cuid()) title String @default("無題のジャーナル") messages Json // [{role: "user"|"assistant", content: string}] mood String? // AI分析による感情ラベル isFavorite Boolean @default(false) date DateTime @db.Date userId String }
日付フィールド：DateTime @db.Date 型（PostgreSQL の DATE 型）。API レスポンスでは ISO 文字列になるため、クライアント側では formatDate() で "YYYY-MM-DD" に変換。

Zustand Store（モバイル側）：
plan: "free" | "premium" — Pro/Free のUIゲーティングに使用
todayStatus: {mood: bool, selfcare: bool, journal: bool} — タブバッジ + クイックアクション完了状態
onboardingDone: bool — オンボーディング完了フラグ（SecureStore と同期）
